101.cherry-pick 순서
    git switch master(master 브랜치로 이동)
    git fetch origin(원격 브랜치 최신화)
    git log origin/codex/modify-search.py-to-count-files -- search.py (codex 브랜치에서 커밋 해시 확인)
       -> 예시 a1b2c3d Update search.py to count files
               e4f5g6h Refactor search logic
    git cherry-pick a1b2c3d(cherry-pick 실행) 
       -> git cherry-pick a1b2c3d^..e4f5g6h(연속된 커밋 범위)
       -> git cherry-pick a1b2c3d e4f5g6h(커밋 여러 개 지정)
    git push origin master(원격 master에 반영)
102 cherry-pick 충돌시 처리
    충돌이 발생된 파일에서 (<<<<<<<, =======, >>>>>>>) 코드 있는 확인 없으면 다음 진행 
    git status
    git add search.py
    git cherry-pick --continue
